# Contributing to Dotfiles

感谢你关注我的配置! 虽然这些配置文件是根据我个人的习惯定制的, 但我非常欢迎修复 Bug, 改进性能或优化文档的贡献

## 🛠 如何贡献
### 1. 报告问题
* 如果你发现某个配置在特定版本的软件上失效了, 请提交 Issue
* ​请注明你的操作系统, Shell 版本以及相关的错误日志

### 2. 提交改进
如果你想直接修改代码, 请遵循以下步骤
1. **Frok** 本仓库
2. 创建一个新的分支 (git checkout -b feature/AmazingFeature)
3. **注意**: 请确保你的改动是**通用性**的, 或者是可以通过环境变量配置的, 我不希望引入仅适用于你个人路径的硬编码
4. 提交你的改动 (git commit -m 'feat: add some cool alias')
5. 推送到分支 (git push origin feature/AmazingFeature)
6. 开启一个 Pull requests

## ⚠️ 注意事项
* **不要提交私密信息**: 请检查你的提交中是否包含 API Keys, 密码, 邮箱或真实的 SSH 密钥
* **保持整洁**: 尽量保持注释清晰, 方便我 (以及其他使用者) 理解你的改动
* **脚本安全**: 如果你修改了安装脚本 (setup.sh), 请务必在你的机器上多次测试

## 💬 灵感与讨论
如果你有一个超棒的插件推荐或配置灵感, 但不确定是否适合我的仓库, 欢迎直接在 Issue 里留言讨论, 不一定非要写代码!
